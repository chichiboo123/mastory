# Mastory UI/UX/배포 점검 보고서

## 1) 핵심 이슈

### UI 안정성
- `selectedSceneId` 상태가 선언되지 않았는데 참조되어 타입체크/빌드가 실패할 수 있는 코드가 존재했습니다.

### UX 사용성
- 상단 데이터/언어 드롭다운이 바깥 영역 클릭으로 닫히지 않아 모바일/데스크톱 모두에서 조작 피로가 있었습니다.
- `Esc` 키로 도움말/내보내기/메뉴를 닫을 수 없어 키보드 접근성이 부족했습니다.
- 일부 주요 피드백(토스트, 빈 상태, 도움말)이 한국어로 하드코딩되어 언어 전환 시 일관성이 깨졌습니다.
- 아이콘 버튼에 접근성 라벨이 부족했습니다.

### 배포 운영성
- GitHub Pages 배포 워크플로우가 `attached_assets/**` 변경을 감지하지 못해, 이미지 교체 시 자동 배포가 누락될 수 있었습니다.

## 2) 적용한 해결 내용

- 빌드 저해 코드(`selectedSceneId` 참조 블록) 제거.
- 외부 클릭 + `Esc` 키 처리 추가로 메뉴/모달 닫기 UX 개선.
- 다국어 키 확장(토스트, 빈 상태, 도움말, 로딩/삭제 라벨) 및 하드코딩 텍스트 치환.
- 상단 아이콘 버튼(`데이터`, `언어`, `도움말`)에 `aria-label` 추가.
- GitHub Pages 워크플로우 `paths`에 정적 자산/의존성 메타 파일 변경 경로 추가.

## 3) 기대 효과

- 빌드 안정성 개선 (타입/런타임 오류 가능성 감소)
- 언어 전환 일관성 및 접근성 향상
- 모바일/데스크톱 인터랙션 피로 감소
- 이미지/자산 변경 시 배포 누락 리스크 감소
